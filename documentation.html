Github

From Branch to Pull Request

Branch

Before any code can be written, a workspace must be created. To do so, navigate to the project you wish to contribute to and create a new branch. Creating a new branch involves choosing a current branch off which to branch and selecting a name for the new branch; these should both follow the project specific guidelines which should be outlined in the readme. Once you have figured out a parent branch and decided on a name for your new branch, open the dropdown (on the left, under the number of commits) and select the branch you wish to branch from. Now, select that dropdown again and begin typing the name of the branch you wish to create. Click "Create branch" and the branch has successfully been created.


Clone

The files from the created branch must now be brought locally onto the development machine. Using git, repositories can be copied locally using clones. When a repository is cloned, it will create a directory with the name of the repository. For example, if your current directory is ~/code and you clone a repository named Simulator, git will create a directory inside ~/code named Simulator and place all the files from the repository inside ~/code/Simulator. 

$ git clone https://github.com/100Hackers/Simulator.git


Checkout

Now that the repository has been setup, change directory into the repository. From inside the directory, the branch created on Github can be tracked. 

$ git checkout --track origin/Simulator


Commit

The files from the created branch must now be brought locally onto the development machine. 

$ git add .
$ git commit -m "Updated collision detection to be Sweep and Prune"


Push

The files from the created branch must now be brought locally onto the development machine. 

$ git push -u origin Simulator
